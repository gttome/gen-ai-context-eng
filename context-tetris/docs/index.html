Priority Ladder (Score Lens)

  1. Non‑negotiables — Role + Rules
  2. Dynamic facts + grounding — keep it relevant
  3. Memory — concise continuity (not raw history)
  4. Examples + checks + schema — 1–2 examples, force structure
  5. User request — place last so it’s the focus
  6. Nice‑to‑have — only if budget remains
Quick fixes (when you overload)
  • Prune noise
  • Summarize memory
  • Retrieve short excerpts (no doc dumping)
  • Structure with headings/checklists
