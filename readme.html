Gen AI Context Engineering

This repository contains exercises and labs focused on context engineering for Large Language Models.

📁 Repository Structure

Chapter 1: Foundations

All Chapter 1 applications are organized within the chp1 directory.

Ordering note: The list below is arranged to follow the Chapter 1 content flow: definition → tokens/window → context toolkit → core patterns → iteration → failure modes → practice.
  • Approach Switchboard: Decision guide for choosing Basic Prompting vs Context Engineering vs RAG vs Fine-Tuning.
  • Context Tetris: Token budgeting and managing the context window as finite working memory.
  • Context Skeleton Studio: Build a reusable context template (role, rules, grounding, memory, schema, request).
  • RAG Snippet Surgeon: Retrieval grounding with short, relevant excerpts (avoid “document dumping”).
  • Memory Mixer: Controlled memory across turns (rolling summary, pinned facts, retrieval memory).
  • Dynamic Facts Firewall: Real-time data injection via Dynamic Facts; keep rules separate from changing values.
  • Iteration Lab: Design → Test → Evaluate → Adjust iteration loop with regression checks.
  • Failure Mode Clinic: Diagnose common failure symptoms and apply quick fixes.
  • Enterprise Scenario Arcade: End-to-end practice mini-scenarios with structured outputs.
